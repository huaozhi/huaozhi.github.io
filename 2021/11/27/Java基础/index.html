Java基础

2021-11-27 Java 1.7k 字
Java

Java特性

  1. Java语言是面向对象的（oop）
  2. Java语言具有稳健性，其强类型机制和垃圾回收机制
  3. Java语言跨平台性
  4. Java是解释型语言

Java核心机制-Java虚拟机

  • JVM是一个虚拟的计算机，具有指令集并使用不同的存储区域，负责执行指令，管理数据、内存、寄存器，包含在JDK中
  • 不同的平台有不同的虚拟机
  • JVM机制屏蔽了底层运行平台的差异，实现了“一次编译，到处运行”

什么是JDK和JRE

JDK

  • JDK 的全称(Java Development Kit Java开发工具包)
    JDK = JRE + java 的开发工具 [java, javac,javadoc,javap 等]
  • JDK 是提供给 Java 开发人员使用的，其中包含了 java 的开发工具，也包括了 JRE。所以安装了 JDK，就不用在单独
    安装 JRE 了。

JRE

  • JRE(Java Runtime Environment Java运行环境)
    JRE = JVM + Java 的核心类库[类]
  • 包括 Java 虚拟机(JVM Java Virtual Machine)和 Java 程序所需的核心类库等，如果想要运行一个开发好的 Java 程序，
    计算机中只需要安装 JRE 即可。

JDK、JRE 和 JVM 的包含关系

  • JDK = JRE + 开发工具集（例如 Javac,java 编译工具等)
  • JRE = JVM + Java SE 标准类库（java 核心类库）

Java开发细节

  • 一个源文件可以有多个类，但是最多只能有一个public类

  • 一个源文件如果有一个public类，类名要与文件名一样

  • Java应用程序的执行入口是main()方法，他有固定的格式：

    public static void main(String[] args) {}

  • main()方法可以写在非public类中

Java转义字符

  • \n ：换行符

  • \r：回车符

    System.out.println("abcde\rfg");输出结果是fgcde

  • \t：一个制表位

  • \\：一个\

  • \'：'

Java注释（comment）

  • 单行注释： //

  • 多行注释：/* 注释内容 */

  • 文档注释：

    • 注释的内容可以被JDK提供的工具javadoc所解析，生成一套以网页文件形式体现的该程序的文档说明一般写在类

    • 基本格式

      /**
      	* @author 小明 
      	* @version 1.0
      */

    • 生成方式：javadoc -d 目标文件夹名 -xx -yy Java文件

      javadoc -d d:/temp -author -version hello.java

    • 上面两个注释内容是javadoc标签

Java代码规范

  • 类、方法的注释，要以javadoc方式来写
  • 运算符和=习惯在两侧加空格
  • 源文件使用utf-8编码
  • 推荐行尾风格

程序中 + 的使用

  • 当左右两边都是数值型，做加法运算
  • 当左右两边有一方是字符串，做拼接运算
  • 运算顺序：从左往右
    • System.out.println(3 + 5 + "abc"); //8abc
    • System.out.println("abc" + 3 + 5); //abc35

编码

韩顺平笔记的第77页

  • ASCLL码
    • 一个字节，最高位0，只用到128个字符
  • Unicode码
    • 两个字节，将世界上所有字符包括起来，没有乱码问题
    • Unicode兼容ASCLL码
  • UTF-8码
    • 使用最广泛，变长编码方式
      • 汉字占3个字节
      • 字母占1个字节
  • gbk编码
    • 一个字母一个字节
    • 一个汉字两个字节

自动类型转换

  • 数据类型按精度（容量）从小到大排序为

    char ==> int ==> long ==> float ==> double

    byte ==> short ==> int ==> long ==> float ==> double

  • 自动类型转换注意和细节

    1. 有多种数据类型的数进行运算时，系统首先自动将所有数据转换成容量最大的那种数据类型，然后再进行计算
    2. 当我们把容量（精度）大的数据类型赋值给容量低的数据类型时会报错，反之就会进行自动类型转换
    3. （byte、short）和 char 不会进行类型转换
    4. byte 、 short 或int 能进行计算，但是会先全部转换成int，出现三者的一个，结果就会转换成int
    5. boolean 不参与自动转换
    6. 给byte型赋值整数常量时，只要数在-128到127就行
    byte n1 = 1;
    byte n2 = 2;
    byte n3 = n1 + n2; // 报错，int -> byte
    byte n4 = 128; // 报错，int -> byte

强制类型转换

细节

  1. 当数据类型从 大 --> 小，就要进行强制类型转换
  2. 强制类型转换符只对最近的操作数起作用，可以用()改变优先级
  3. char 类型可以保存int常量值（未溢出2字节），但不能保存int变量值
  4. byte ,short,char运算时都转换成int

算数运算

i = 1;
i = i ++; // temp = i; i = i + 1; i = temp; 输出1
i = 1;
i = ++ i; // i = i + 1; temp = i; i = temp; 输出2

赋值运算

a += b <==> a = a + b// 复合赋值运算符会有隐含强制类型转换byte a = 1;a++; // a = (byte)(a + 1);a += 2; // a = (byte)(a + 2);

键盘输入

需要用到Scanner类，在java.util.*

import java.util.Scanner;Scanner myScanner = new Scanner(System.in);System.out.println("请输入名字");String name = myScanner.next(); //接收用户输入字符串System.out.println("请输入年龄");int age = myScanner.nextInt(); //接收用户输入 intSystem.out.println("请输入薪水");double sal = myScanner.nextDouble(); //接收用户输入 double

原码、补码、反码

  • 正数三码合一
  • 负数的反码是符号位不变，各位取反；补码是反码加一
  • 0的补码和反码都是0
  • Java没有无符号数，所有的数都是有符号的
  • 计算机运算的时候都是以补码形式运算的

位运算符

  • 按位与&
  • 按位或|
  • 按位异或^
  • 按位取反~
  • 算数右移>>：低位溢出，高位用符号位补
  • 算数左移<<：高位溢出，低位用0补
  • （无符号右移）逻辑右移>>>：低位溢出，高位用0补
  • 没有<<<
  • 位运算用补码进行

进制

  • 二进制： 以0b或0B开头， 0b10010
  • 八进制： 以0开头 ， 010010
  • 十进制：12
  • 十六进制： 以0x或0X开头， 0XAF
