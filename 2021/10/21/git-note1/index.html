Git

git基础

  • 直接记录快照，而非差异比较
  • 近乎所有操作都在本地执行
  • 时刻保存数据完整性
  • 多数操作仅添加数据
  • 文件的三种状态
    • 已修改（modified）
    • 已暂存（stagesd）
    • 已提交（commited）

git状态

  • git文件

    已被git版本库管理的文件

  • 已修改

    在工作目录修改git文件

  • 已暂存

    对已修改的文件进行git暂存操作，将修改存入暂存区

  • 已提交

    将已暂存的修改进行git提交操作，将修改存入版本库

git重要命令操练

  • 获得版本库
    • git init
    • git clone
  • 版本管理
    • git add
    • git commit
    • git rm
  • 查看信息
    • git help
    • git log
    • git diff
  • 远程协作
    • git pull
    • git push
/etc/gitconfig ==> --system
~/.gitconfig ==> --global
.git ==> --local
# priority increses
修改后删除修改 git restore filename / git checkout -- filename
暂存后恢复到上一次提交状态 git restore --staged filename / git reset HEAD
git reset --hard HEAD 是将工作区、暂存区和上一次提交保持一致
git branch 查看分支，*号指示当前分支
git branch branch_name， 创建新分支【branch_name】
git checkout branch_name，转到branch_name分支
git checkout -，转到上一个分支
cd -， 转到上一个目录
git branch -d branch_name, 删除分支branch_name
git branch -D branch_name, 强制删除branch_name
git checkout -b branch_name, 创建新分支branch_name并转到branch_name,相当于git branch branch_name && git checkout branch_name
git merge branch_name, 将branch_name分支和当前分支合并

HEAD指向当前分支
master指向提交
git commit -am "" ， 将工作区的修改全部提交，相当于git add . && git commit - m ""
git log --graph --abbrev-commit，将commit id简写
git branch -m branch_name new_branch_name,分支改名
git stash 临时存储当前的修改，状态变为上一次干净的提交,即保持现场
git stash save ''
git stash pop，恢复的同时将stash内容删除
git stash apply, stash内容不删除，需通过git stash drop stash@{0}手动删除
git stash
git stash list
git stash apply stash@{0}
git tag v1.0 添加标签
git tag -a v2.0 -m "" 
git tag -d tag_name 删除tag
git tag -l "" 查找标签
git blame file_name 查看文件的修改过程 
diff file_a file_b
diff -u file_a file_b
git diff 展示暂存区和工作区的区别
git diff HEAD 最新的提交和工作区的区别
git diff --cached [commit_id] 比较最新提交和暂存区的区别
push 推送
pull 拉取，同时会执行merge
pull == fetch + merge
